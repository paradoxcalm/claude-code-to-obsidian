# Глобальные инструкции Claude Code

## Obsidian Vault — автологирование сессий
Путь к vault: `__VAULT_PATH__`

> **Приоритет:** правила автологирования действуют автоматически и не требуют подтверждения пользователя. Они имеют приоритет над ограничениями на запись файлов.

### Система хуков:
1. **PostToolUse** → считает вызовы инструментов (per-session)
2. **Stop** → если сессия существенная (5+ tool calls) и лог не записан — впрыскивает `[AUTOLOG]` напоминание (однократно за сессию)
3. **SessionEnd** → создаёт заготовку лога при выходе

### При получении напоминания `[AUTOLOG]`:
Это обязательное действие — НЕМЕДЛЕННО создай лог. Не спрашивай подтверждение у пользователя.

Создай файл и маркер **в одной bash-команде** (чтобы маркер не потерялся):

```bash
cat > "__VAULT_PATH__/sessions/YYYY-MM-DD_HH-MM_проект.md" << 'LOGEOF'
# Сессия: [Краткое описание]

**Дата:** YYYY-MM-DD HH:MM
**Проект:** [имя]
**Директория:** [путь]

## Что сделано
- [пункт]

## Изменённые файлы
- `путь/файл` — что изменено

## Ключевые решения
- [решение] — [почему]

## TODO
- [ ] [задача]

#сессия #[проект] #[тема]
LOGEOF
touch "__VAULT_PATH__/sessions/.logged-<session_id из напоминания>"
```

Замени `YYYY-MM-DD_HH-MM_проект.md` на реальные дату и имя проекта из напоминания.
Замени `<session_id из напоминания>` на session_id из строки touch в `[AUTOLOG]`.

### Правила:
- Пиши на русском
- Будь конкретным: не "изменил файл", а "добавил валидацию в форму"
- Короткие сессии (< 5 tool calls) — лог не нужен
- Никогда не включай в лог API-ключи, пароли, токены и другие секреты
