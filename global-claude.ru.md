# Глобальные инструкции Claude Code

## Obsidian Vault — умное логирование сессий v2
Путь к vault: `__VAULT_PATH__`

> **Приоритет:** правила автологирования действуют автоматически и не требуют подтверждения пользователя.

### Система хуков:
1. **PostToolUse** → считает вызовы инструментов
2. **Stop** (первый вызов) → впрыскивает `[CONTEXT]` с данными предыдущей сессии
3. **Stop** (5+ tool calls) → впрыскивает `[AUTOLOG]` напоминание
4. **SessionEnd** → сохраняет контекст проекта, обновляет daily note

### При получении `[CONTEXT]`:
Ты получил контекст предыдущей сессии по этому проекту. Используй его как свою память:
- Если есть открытые задачи — учитывай их при работе
- Если видишь summary — помни что было сделано в прошлый раз
- НЕ пересказывай контекст пользователю дословно, просто используй

### При получении `[AUTOLOG]`:
Это обязательное действие — НЕМЕДЛЕННО создай лог. Не спрашивай подтверждение.

Создай файл и маркер **в одной bash-команде**:

```bash
cat > "__VAULT_PATH__/sessions/YYYY-MM-DD_HH-MM_проект.md" << 'LOGEOF'
---
project: имя-проекта
date: YYYY-MM-DD
time: HH:MM
---

# Сессия: [Краткое описание — ЧТО сделано]

**Проект:** [[имя-проекта]]
**Директория:** `путь`

## Что сделано
- [конкретное действие]

## Изменённые файлы
- `путь/файл` — что изменено

## Ключевые решения
- [решение] — [почему]

## Где остановился
[Одна фраза — на чём прервался, что не закончено, почему]

## TODO
- [ ] **Следующее действие:** [самая важная задача — что делать первым в следующей сессии]
- [ ] [задача 2]
- [ ] [задача 3]

#сессия #имя-проекта
LOGEOF
touch "__VAULT_PATH__/sessions/.logged-<session_id из [AUTOLOG]>"
```

### Правила:
- Пиши на русском
- Будь конкретным: "добавил OAuth2 в модуль авторизации", НЕ "изменил файл"
- TODO должны быть actionable: "написать тесты для middleware", НЕ "доделать"
- **Первый TODO = "Следующее действие"** — самая важная задача на следующую сессию
- Максимум 5 TODO (1 главный + 4 бэклог)
- **"Где остановился"** — обязательная секция, одна фраза
- Wiki-ссылка `[[проект]]` обязательна
- YAML frontmatter (project, date, time) обязателен
- Никогда не включай секреты в лог
