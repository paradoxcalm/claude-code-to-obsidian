# Claude Code → Obsidian: Навыки работы с vault

## Чтение логов
- При входе в проект определи имя из basename текущей директории
- Найди файлы в `__VAULT_PATH__/sessions/` содержащие это имя проекта в frontmatter `project:`
- Прочитай последние 3 по дате
- Извлеки: незакрытые TODO, секцию "Где остановился", ключевые решения
- Покажи пользователю в 3 строки: "Последняя сессия: {дата}. Остановился на: {фраза}. Открытых TODO: {число}"

## Запись логов
- Всегда используй YAML frontmatter: project, date, time, tags, files_changed, status, previous_session
- status: "in-progress" если есть незакрытые TODO, "completed" если все закрыты
- tags в frontmatter = массив: [auth, database, bugfix] и т.д.
- Первый TODO всегда помечай как "Следующее действие"
- Максимум 5 TODO (1 главный + 4 бэклог)
- files_changed — целое число изменённых файлов за сессию
- previous_session — ссылка на предыдущий лог этого проекта (из [CONTEXT] если доступен)

## Связывание
- Если работа затрагивает другой проект — добавь wiki-ссылку [[другой-проект]]
- Если повторяется баг из прошлой сессии — ссылайся: "См. [[sessions/2024-03-10_my-app]]"
- В секции "Ключевые решения" ссылайся на технологии: [[PostgreSQL]], [[OAuth2]]
- Используй `previous_session` в frontmatter для создания цепочки сессий

## Обновление проектного контекста
- После записи лога — проверь файл `__VAULT_PATH__/projects/{project-name}.md`
- Если файл существует — данные обновляются автоматически хуком SessionEnd:
  - Последняя сессия: дата и ссылка
  - Текущий статус: что в процессе
  - Открытые TODO: собранные из всех sessions/ по этому проекту
  - Стек: технологии определяются по расширениям изменённых файлов
- Контекст кэшируется в `__VAULT_PATH__/projects/.context-{project-name}.json`
- При следующей сессии хук Stop автоматически впрыснет [CONTEXT] из этого кэша
