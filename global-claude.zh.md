# Claude Code 全局指令

## Obsidian Vault — 智能会话日志 v2
Vault 路径: `__VAULT_PATH__`

> **优先级:** 自动日志规则自动生效，无需用户确认。

### 钩子系统:
1. **PostToolUse** → 统计工具调用次数
2. **Stop**（首次调用）→ 注入 `[CONTEXT]` 上次会话数据
3. **Stop**（5+ 次工具调用）→ 注入 `[AUTOLOG]` 提醒
4. **SessionEnd** → 保存项目上下文，更新每日笔记

### 收到 `[CONTEXT]` 时:
你收到了该项目上次会话的上下文。将其作为你的记忆使用：
- 如果有未完成的任务 — 在工作中考虑它们
- 如果看到摘要 — 记住上次做了什么
- 不要向用户逐字复述上下文，直接使用

### 收到 `[AUTOLOG]` 时:
这是必须执行的操作 — 立即写入日志。不要请求确认。

用**一条 bash 命令**同时创建文件和标记：

```bash
cat > "__VAULT_PATH__/sessions/YYYY-MM-DD_HH-MM_项目名.md" << 'LOGEOF'
---
project: 项目名
date: YYYY-MM-DD
time: HH:MM
---

# 会话: [简要描述 — 做了什么]

**项目:** [[项目名]]
**目录:** `路径`

## 完成内容
- [具体操作]

## 变更文件
- `路径/文件` — 具体变更

## 关键决策
- [决策] — [原因]

## 停止位置
[一句话 — 中断了什么，什么未完成，原因]

## TODO
- [ ] **下一步操作：** [最重要的任务 — 下次会话首先做什么]
- [ ] [任务 2]
- [ ] [任务 3]

#会话 #项目名
LOGEOF
touch "__VAULT_PATH__/sessions/.logged-<[AUTOLOG] 中的 session_id>"
```

### 规则:
- 用中文撰写
- 要具体："为认证模块添加了 OAuth2"，而不是"修改了文件"
- TODO 必须可执行："为 middleware 编写测试"，而不是"继续完成"
- **第一个 TODO = "下一步操作"** — 下次会话最重要的任务
- 最多 5 个 TODO（1 个主要 + 4 个待办）
- **"停止位置"** — 必填部分，一句话
- Wiki 链接 `[[项目]]` 必须包含
- YAML frontmatter（project, date, time）必须包含
- 永远不要在日志中包含敏感信息
